Email resources enable a web service call to send emails using the credentials entered into the configuration.

INodemailer is the underlying library and the configuration, request, and response will reflect that library.


GET (get file)

Request 
{{resource.path.replace('*','')}}/:filename
Response 
the file

POST (upload file)

Request 
must have the header Content-Type: multipart/form-data
Response 
the file path

PUT/PATCH (rename file)

Request 
{ name: 'newfilename.ext' }
Response 
the new file path ex./ {{resource.path.replace('*','')}}/newfilename.ext

DELETE (remove file)

Request 
{{resource.path.replace('*','')}}/:filename
Response 
success/error message
