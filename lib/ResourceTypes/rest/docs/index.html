A rest resource behaves as a proxy or pass-through to another rest api that you want to integrate into hubba.

All http verbs are passed through to the configured endpoint, and the configuration allows the option to pass-through or not pass-through headers, url parameters, query string parameters, and body data.

If all the options are checked a GET/POST/... request to '{{resource.path.replace('*','')}}' would be exactly the same as a GET/POST/... request to '{{resource.configuration.url}}' the configured url.

Any query parameters or headers entered into the configuration would override what comes from the request.
