Controller resources are meant to mimic the common definition of a controller in a web framework.

Custom code can be written for each basic HTTP verb (GET,POST,PUT,PATCH,DELETE) that will respond to requests.

Controllers can call other controllers using the hubba.api object.

See below for details for each accessible object from within a controller

hubba

The hubba client is accessible via the controller and will mimic a web service call to another web service.

Documentation on the hubba client can be found here

req

This object is modeled off of the expressjs request object: documentation here

The following fields/functions from expressjs are available:

  • contentLength
  • contentType
  • body
  • href
  • log
  • id
  • path
  • query
  • secure
  • time
  • header

The following fields/functions are unique to hubba:

  • internal - this field will be true if the request came from another controller

res

This object is modeled off of the expressjs response object: documentation here

The following fields/functions from expressjs are available:

  • status
  • set
  • get
  • cookie
  • clearCookie
  • redirect
  • location
  • charset
  • send
  • json
  • jsonp
  • type
  • format
  • attachment
  • sendfile
  • download
  • links
